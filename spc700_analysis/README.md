# SPC700程序代码分析

## 概述

反汇编SPC700 RAM中的程序代码，分析Metal Max Returns音频引擎的结构。

## 分析结果

| 指标 | 值 |
|------|-----|
| 分析的SPC转储 | 4 (title/intro/town/wm) |
| 复位向量 | $8178 (所有场景一致) |
| 入口点指令数 | 279 |
| 子程序数 | 7 |
| 端口通信点 | 1 |

## SPC700音频引擎结构

### 入口点 ($8178)
所有场景的SPC700程序入口点相同（$8178），程序代码在场景间不变化。音频数据（BRR样本）会随场景变化，但引擎代码是固定的。

### 子程序
通过分析CALL指令目标，识别出7个子程序调用点。子程序地址分布在$0005-$00BD范围，部分位于低地址区域（可能是数据区被误识别）。

### DSP寄存器访问
SPC700通过I/O端口$F2（寄存器选择）和$F3（数据写入）控制DSP：
- 写入$F2选择DSP寄存器地址
- 写入$F3设置该寄存器的值

### SNES通信端口
SPC700与SNES通过端口$F4-$F7进行通信：
- SNES向SPC700发送命令（如播放曲目、切换音效）
- SPC700向SNES返回状态

## SPC700指令集特点
- 8位架构，与6502类似但有显著差异
- 累加器A、索引寄存器X/Y
- YA寄存器对（16位操作）
- 支持直接页寻址、绝对寻址、变址寻址
- 专用指令：MUL YA（乘法）、DIV YA X（除法）、DAA/DAS（BCD调整）

## 技术限制

- 线性反汇编无法区分代码和数据，部分数据区会被误反汇编
- 需要完整的代码流分析才能准确识别所有子程序
- DSP寄存器写入模式检测基于固定字节序列匹配，可能遗漏
