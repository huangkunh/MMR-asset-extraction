# ROM数据表提取

## 概述

从Metal Max Returns的ROM数据bank中提取结构化数据表，包括重复记录表、指针表、字符串和数值表。

## 提取结果

| 类型 | 数量 | 说明 |
|------|------|------|
| 真实数据表 | 1,213 | 过滤空数据后的有效重复记录结构 |
| 有效指针表 | 11,226 | 指向非空数据的LoROM地址指针表 |
| 数据Bank字符串 | 3,159 | SJIS编码的文本片段 |
| 数值数据表 | 3,165 | 小数值(0-100)连续区域 |

## 关键发现

### 指针表 @ 0x029BE4
- 100个指针，全部指向非空数据
- 指针范围: $9B0A-$9D75
- 指向的数据为65816汇编代码，可能是函数调用表/事件跳转表

### 属性数据表 @ 0x05C5A8 (Bank $0B)
- 30条记录，每条12字节
- 字节值范围0-255，平均值8-35
- 可能是角色/敌人属性表（HP、攻击力、防御力等）

### 代码bank分布
- Bank $04-$0B: 混合数据和代码
- 大量指针表集中在Bank $05 (offset 0x028000-0x030000)
- Bank $0B包含结构化数值数据

## 文件说明

| 文件 | 说明 |
|------|------|
| `rom_tables_report.txt` | 原始提取报告（含空数据） |
| `rom_tables_report.json` | JSON格式报告 |
| `rom_tables_deep_analysis.txt` | 深度分析报告（过滤空数据） |
| `rom_strings.txt` | 数据Bank字符串提取 |

## 扫描范围

- ROM大小: 4,194,816 bytes (4MB)
- 扫描bank: $04-$0B (8个数据bank, 64KB)
- LoROM映射: bank × $8000 + (addr & $7FFF)

## 数据表识别方法

1. **重复记录结构**: 固定长度记录，字节级相似度>20%
2. **指针表**: 连续16位LoROM地址($8000-$FFFF)
3. **数值表**: 连续区域大部分值在0-100范围内
4. **字符串表**: SJIS双字节字符序列
